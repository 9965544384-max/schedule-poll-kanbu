# 日程調整ツール

時間範囲をドラッグで一括指定 × ◯△☓で回答できる、調整さん風 × When2meet風のハイブリッド日程調整ツール。

## セットアップ手順

### 1. このフォルダの中身をGitHubリポジトリにアップロード

GitHubで新しいリポジトリ（例: `schedule-poll`）を作成し、以下のファイルをすべてアップロード：

- `index.html`
- `app.js`
- `style.css`
- `config.js`
- `README.md`（任意）

### 2. GitHub Pagesを有効化

1. リポジトリページで **Settings** タブをクリック
2. 左サイドバーの **Pages** をクリック
3. **Source** で `Deploy from a branch` を選択
4. **Branch** で `main`（または `master`）と `/ (root)` を選んで **Save**
5. 1〜2分待つと、ページ上部に `Your site is live at https://[ユーザー名].github.io/schedule-poll/` と表示される

### 3. 完成

そのURLを開けば、調整さん風のツールが動きます。共有URLをLINEなどで送れば、誰でも◯△☓で回答できます。

## 使い方

### 主催者
1. トップページで「新しい調整を作成」をクリック
2. イベント名・日付範囲・時間範囲を指定して作成
3. 表示された共有URLをLINEなどで参加者に送る

### 参加者
1. 共有URLを開く
2. 名前を入力
3. ◯△☓ボタンを選んでセルをドラッグまたはクリックで回答
4. 「保存」を押す

### 集計確認
- 集計画面で各時間枠の回答状況がヒートマップで見られる
- 「参加できる人が多い時間 トップ5」が自動表示
- 「↻ 更新」を押すと最新の回答を取得

## 技術スタック

- **フロントエンド**: 素のHTML/CSS/JavaScript（フレームワークなし）
- **データベース**: Supabase（無料枠）
- **ホスティング**: GitHub Pages（無料）

## ファイル構成

```
schedule-poll/
├── index.html      # エントリーポイント
├── app.js          # メインロジック
├── style.css       # スタイル
├── config.js       # Supabase接続設定
└── README.md       # このファイル
```

## カスタマイズ

`style.css` の `:root` セクションでテーマカラーを変更できます。
`app.js` の最大日数（60日）や時間刻みなどはコード内の定数を編集してください。
